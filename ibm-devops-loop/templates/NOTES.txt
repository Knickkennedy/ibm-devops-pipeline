Known issue: In some cases the password used by RabbitMQ is not correctly initialized. This causes
some pods to fail to connect and then restart. To correct the password manually, use the command:
 $ {{ .Values.global.cmdKubectl }} exec -n {{ .Release.Namespace }} {{ .Release.Name }}-rabbitmq-0 -- rabbitmqctl change_password user \
    "$({{ .Values.global.cmdKubectl }} get secret -n {{ .Release.Namespace }} {{ .Release.Name }}-rabbitmq -o jsonpath='{.data.rabbitmq-password}' | base64 --decode)"
